Disney SD Smart Vertical Streaming — Project Documentation
==========================================================

PROJECT OVERVIEW
----------------
Disney Senior Design Project: Smart Vertical Basketball Streaming
Primary goal: Automatically reframe landscape basketball video to 9:16 vertical format
using YOLOv8 detection + ByteTrack multi-object tracking (via ultralytics).

Three core features are exposed through the Streamlit UI:
  1. Player Detection  — detects and tracks people with bounding boxes + IDs
  2. Ball Detection    — detects the basketball using YOLO + Gaussian filter smoothing
  3. Smart Crop        — auto-reframes the video to 9:16 following the ball

FILES USED BY THE UI
--------------------
All detection logic has a single source of truth. UI/detection.py pulls symbols
from the standalone scripts via importlib — it does not re-implement anything.

  UI/StreamJup.py               — entry point
  UI/detection.py               — processing engine (consumer of all modules below)
  UI/html_builders.py           — HTML/JS video players with canvas overlays
  UI/video_utils.py             — video encoding and HTTP serving

  cropping/smartCroppingVideos.py
    — loaded by detection.py for: cropped_width, cropped_height, BallTracker

  detection/ball-detection/ball_detection.py
    — loaded by detection.py for: GAUSSIAN_SIGMA, _choose_best_ball,
      _is_valid_ball_size, _reject_spatial_outliers

  detection/player-detection/player_detection.py
    — loaded by detection.py for: draw_tracked_boxes

NOT USED BY THE UI (standalone only / groundwork for future expansion):
  cropping/smartCroppingImages.py
    — Groundwork for expanding into image-level smart cropping.
  cropping/reframe_9x16.py
    — Groundwork for an alternative player-motion-based crop mode.
  cropping/read-and-crop.py
    — Simple centered crop baseline utility. No AI.
  tests/
    — Test suite. Uses kagglehub to pull real basketball footage for testing.

UI MODULE BREAKDOWN
-------------------
StreamJup.py
  - Handles file upload, session state, button layout, metrics display, and tabs.
  - Calls the three process_* functions from detection.py on button click.
  - Loads YOLOv8m model once at startup via @st.cache_resource.
  - Uses html_builders.py to embed the video player into an iframe.
  - Uses video_utils.py to encode and serve the processed video.

detection.py
  - Single import point for all detection logic. Uses importlib to load three
    external modules at startup (smartCroppingVideos, ball_detection,
    player_detection) so UI code only calls functions — no logic is duplicated.
  - process_video(video_path, model)
      Runs YOLOv8 + ByteTrack on every frame (classes=[0], person only).
      Returns clean frames, annotated frames, per-frame counts, bounding boxes,
      video dimensions, fps, and elapsed time.
  - process_ball_video(video_path, model)
      2-pass pipeline: detects sports ball (COCO class 32) using _choose_best_ball
      and _is_valid_ball_size, rejects outliers with _reject_spatial_outliers,
      gap-fills and Gaussian-smooths (sigma=15) the ball center trajectory.
      Returns clean frames, per-frame ball boxes, smoothed positions, counts,
      video dimensions, fps, and elapsed time.
  - process_smart_crop_video(video_path, model)
      Runs the same ball detection and smoothing pipeline, then derives a
      Gaussian-smoothed crop window from the ball center. Crops and resizes
      each frame to 540x960. Returns cropped frames, remapped ball boxes,
      smoothed center coords, fps, and elapsed time.

html_builders.py
  - build_player_html(url, frame_boxes, frame_w, frame_h, fps)
      HTML5 video + canvas overlay with requestAnimationFrame loop.
      Player bounding box data embedded as JSON; toggled via button.
  - build_ball_html(url, frame_ball_boxes, frame_gaussian, frame_w, frame_h, fps)
      Same structure; draws ball detection box and Gaussian-smoothed position circle.
      Two independent toggle buttons (raw box / smoothed circle).
  - build_smart_crop_html(url, sc_ball_boxes, sc_pred, fps)
      540x960 video player with ball box and trajectory prediction circle overlays.

video_utils.py
  - render_video(frames, fps, label) -> str
      Encodes a list of RGB frames to a temp H.264 (avc1) MP4 file using
      OpenCV VideoWriter. Returns the file path.
  - _start_video_server(video_path, label) -> str
      Shuts down any existing server for that label, starts a new HTTPServer
      on a random port serving the video directory. Supports HTTP range requests
      so browsers can seek without downloading the whole file.
  - Cleans up all temp MP4 files at process exit via atexit.


STANDALONE SCRIPTS
------------------
detection/ball-detection/ball_detection.py
  - Source of truth for all ball detection helpers (see FILES USED BY THE UI).
  - Run directly to process all videos in videos/ and save annotated output to output/.
  - 2-pass pipeline: detect all frames → outlier rejection → gap-fill → Gaussian smooth.
  - Renders green box (raw YOLO detection) + red circle (Gaussian-smoothed center).

detection/player-detection/player_detection.py
  - Source of truth for draw_tracked_boxes (see FILES USED BY THE UI).
  - Run directly to process all videos in videos/ and save annotated output to output/.
  - Uses ByteTrack (model.track tracker="bytetrack.yaml"), COCO class 0 (person).
  - Shows live cv2 window during processing with per-frame FPS overlay.

cropping/smartCroppingVideos.py
  - Source of truth for crop constants and BallTracker (see FILES USED BY THE UI).
  - Run directly to produce smart-cropped 9:16 output videos and debug overlays.
  - BallTracker uses polynomial trajectory fitting (weighted least-squares, degree 1 or 2)
    and per-frame outlier detection via prediction-vs-actual distance.
  - BallTracker is stateful and frame-by-frame, making it well-suited for future
    adaptation to live stream cropping.

cropping/smartCroppingImages.py
  - Processes still images from the Kaggle DeepSport dataset.
  - Detects ball in each image, saves cropped 540x960 PNGs. Falls back to
    previous frame's crop location when detection fails.
  - NOT used by the UI. Groundwork for expanding into image-based smart cropping.

cropping/reframe_9x16.py
  - Alternative reframing strategy: tracks player movement speeds per ID using
    SaliencySelector class, selects the top-K fastest movers as the crop focus
    instead of following the ball.
  - Depends on yolox package (noted as a migration candidate to ultralytics ByteTrack).
  - NOT used by the UI. Groundwork for an alternative or complementary crop mode.

cropping/read-and-crop.py
  - Simple utility: reads MP4s from a Kaggle dataset, applies a centered 9:16 crop
    (no AI), writes 540x960 MP4s. Useful as a baseline for crop dimension testing.


FUTURE WORK / EXPANSION
------------------------
- The smartCroppingVideos.py and smartCroppingImages.py scripts establish the
  foundation for expanding this project into live stream smart cropping:
    - BallTracker is already stateful and processes frame-by-frame, so it can be
      adapted to a streaming context with minimal changes.
    - smartCroppingImages.py provides the image-level detection fallback strategy
      needed when frames have no ball detection.
    - reframe_9x16.py provides an alternative player-motion-based crop strategy
      that could be plugged in as a third cropping mode or used as a fallback when
      ball detection confidence is low.


SETUP
-----
1. Clone the repository:
      git clone <repo-url>
      cd Disney-SD-Smart-Vertical-Streaming

2. Create the virtual environment:
      python -m venv sd_disney

3. Activate it:
      Windows (PowerShell):  .\sd_disney\Scripts\Activate.ps1
      Windows (CMD):         sd_disney\Scripts\activate.bat
      Mac/Linux:             source sd_disney/bin/activate

4. Install dependencies:
      pip install -r requirements.txt


RUNNING THE UI
--------------
From the project root, double-click or run:
      .\run_ui.bat

This launches UI/StreamJup.py via Streamlit.

If the .bat launcher is broken (stale venv path), run manually:
      & ".\sd_disney\Scripts\python.exe" -m streamlit run UI\StreamJup.py


KEY DEPENDENCIES
----------------
- ultralytics    — YOLOv8 inference + ByteTrack tracker (model.track / model.predict)
- opencv-python  — Video I/O, frame processing, H.264 encoding
- streamlit      — Web UI framework
- numpy          — Array math, interpolation (gap-fill), crop window clamping
- scipy          — Gaussian filtering (ball center smoothing), MAD-based outlier rejection
- torch          — CUDA device detection (used by standalone scripts)
- tqdm           — Progress bars in standalone scripts
- kagglehub      — Dataset download (used by cropping scripts and tests only)


KNOWN ISSUES
----------------------------
- Pylance reports "unknown import symbol" for detection.py imports in StreamJup.py.
  Root cause: the detection/ folder at the project root shadows UI/detection.py
  for the static analyzer. Runtime is unaffected (Streamlit adds UI/ to sys.path).
  Fix: add a pyrightconfig.json at the project root with { "extraPaths": ["UI"] },
  or move detection/ball-detection and detection/player-detection to the root so
  the detection/ folder no longer exists.

- The ABC logo is occasionally misdetected as a sports ball (COCO class 32) by the
  general-purpose yolov8m model. Proposed fixes:
    a) Raise confidence threshold above 0.20
    b) Add a spatial exclusion zone for the logo area
    c) Use a sports-specific fine-tuned YOLO model
    d) Use higher resolution videos and images
